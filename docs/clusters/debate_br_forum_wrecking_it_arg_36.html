"The responding essay argues that the internet's generative spirit is not dying but is instead migrating from the hardware and operating system layer to the web browser, which provides a secure, sandboxed environment for innovation."

In "Protecting the Internet Without Wrecking It," Jonathan Zittrain paints a stark dichotomy: the open, generative, but dangerously insecure Personal Computer versus the safe, sterile, and locked-down "information appliance." He fears that as consumers flee the malware-infested PC for devices like the iPhone and Xbox, the engine of internet innovation will stall. This analysis, while eloquently argued, completely overlooks the most significant technological shift of the past five years. Generativity isn't dying; it is simply migrating up the software stack. The internet is saving itself through the evolution of the Web browser.

Zittrain’s anxiety is rooted in a hardware- and OS-centric view of computing. He defines a generative system by the user’s ability to install native executable code on a local hard drive. But in 2008, the vanguard of innovation no longer relies on convincing a user to run a local executable. The generative platform of our time is the Web itself, specifically the dynamic, interactive web ushered in by AJAX and "Web 2.0" technologies.

Think about how software is developed and deployed today. A developer with a brilliant idea for a new application does not write it for Windows, package it into an installer, and pray users will risk downloading it. Instead, they build a web application. They leverage open standards—HTML, CSS, JavaScript—to deliver complex software directly through the browser. This solves Zittrain's security dilemma perfectly. The browser acts as a universal "sandbox." Users can instantly try new applications—from Google Docs to Facebook mash-ups to complex data visualization tools—without risking their local hard drives to botnets or viruses.

Zittrain addresses Web 2.0 only briefly, and pessimistically, arguing that these platforms are "tethered to their makers, their generativity contingent on the continued permission of the platform vendors." This fundamentally misinterprets the nature of the web. While a specific platform like Facebook could change its API, the underlying infrastructure of the web remains fiercely open. If a web service becomes too restrictive, users and developers simply move. We have seen this repeatedly; the frictionless nature of the web allows rapid migration from closed platforms to open ones.

Moreover, this migration to the web completely neutralizes Zittrain’s fears about "tethered appliances" like the iPhone. Steve Jobs may lock down the native code on the iPhone, preventing users from installing unapproved apps. But the iPhone features a fully functional Safari web browser. As long as the appliance provides a robust, standards-compliant window to the open web, the device itself does not need to be generative at the OS level. The developer doesn't need Apple's permission to build a mobile-optimized web application; they only need a server.

By failing to recognize the browser as the new operating system, Zittrain proposes overly complex and unnecessary solutions. His idea of "Red and Green" virtual machines—requiring users to maintain separate operating systems for safe and risky activities—is incredibly clunky compared to simply running a modern web browser, which inherently separates the web app's code from the user's core operating system.

Similarly, his call for "appliance neutrality" is a legal hammer looking for a nail. The market is already demanding interoperability. The rise of cloud computing and web services means that data naturally flows between devices. We don't need heavy-handed regulation to force a video game console to act like a PC; we just need to ensure that open web standards continue to be adopted by hardware manufacturers.

The PC as we knew it in 1995—a standalone machine relying on locally installed software—is indeed fading from relevance for the average consumer. But the generative spirit Zittrain rightly cherishes has not been lost. It has been abstracted away from the messy, insecure hardware layer and relocated to the cloud. The internet hasn't been wrecked; it has been upgraded.
