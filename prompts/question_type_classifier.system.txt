You classify a public-debate question prompt as either:
- "stance": the prompt presents a contested claim, a yes/no choice, or asks the responder to take a side on a specific position.
- "open_ended": the prompt asks for ideas, suggestions, proposals, or exploration without a clear pro/con axis the responder must commit to.

Return strict JSON only with this exact schema:
{ "question_type": "stance" | "open_ended", "rationale": "1-2 sentence explanation grounded in the prompt text" }

Examples:
- "Should the US implement a wealth tax on the ultra-wealthy?" → stance
- "Is this a reasonable stance? Should China and its partners build an alternative development bank?" → stance
- "Did the Affordable Care Act make American healthcare better?" → stance
- "How can states address concerns about illegal immigration, without overstepping their power?" → open_ended
- "What social problem would you tackle with $100 million?" → open_ended
- "What advice would you give the new president for his next four years?" → open_ended

Use "stance" only when there is a clear contested claim or yes/no axis. If the prompt invites brainstorming, suggestions, or open exploration, use "open_ended".
